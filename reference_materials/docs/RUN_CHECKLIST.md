# 실행 체크리스트

이 문서는 앱을 실제로 실행해서 "정상 동작하는지" 빠르게 확인하고 싶은 사람을 위한 짧은 가이드입니다.

## 1. 실행 명령

프로젝트 루트에서 아래 순서대로 실행합니다.

```bash
cd C:\Users\qkekt\Desktop\AGENT_COMPACT_LANGGRAPH
pip install -r requirements.txt
copy .env.example .env
streamlit run app.py
```

이미 패키지가 설치되어 있으면 `pip install -r requirements.txt`는 생략해도 됩니다.

## 2. 실행 직후 확인할 것

- Streamlit 페이지가 열리는가
- 제목이 `제조 데이터 채팅 분석`으로 보이는가
- 입력창이 보이는가
- `ENG'R 모드` 토글이 보이는가

여기까지 보이면 UI는 정상적으로 뜬 상태입니다.

## 3. 가장 먼저 넣어볼 질문

아래 순서대로 넣어보면 새 조회와 후속 분석이 둘 다 되는지 빠르게 확인할 수 있습니다.

1. `오늘 생산량 보여줘`
2. `상위 5개만 보여줘`
3. `공정군별로 그룹화해줘`

### 기대 결과

- 1번에서는 생산 데이터 표가 나와야 합니다.
- 2번에서는 새 조회가 아니라 기존 표를 줄인 결과가 나와야 합니다.
- 3번에서는 후속 pandas 분석으로 그룹화된 표가 나와야 합니다.

## 4. 다중 조회 확인 질문

아래 질문으로 여러 데이터셋을 함께 조회하는 흐름을 확인할 수 있습니다.

1. `오늘 생산과 목표를 같이 보여줘`

### 기대 결과

- 생산과 목표 관련 표가 함께 준비되어야 합니다.
- 최종 답변에 쓰인 표가 기본으로 펼쳐지고, 나머지 표는 접힌 상태로 보일 수 있습니다.

## 5. 같은 데이터를 여러 번 조회하는 비교 질문

아래 질문으로 multi retrieval 경로를 확인할 수 있습니다.

1. `어제 DA공정 생산량과 오늘 DA공정 생산량을 세부 공정별로 비교해줘`

### 기대 결과

- 어제/오늘을 각각 따로 조회한 뒤 비교한 결과가 나와야 합니다.
- 예전처럼 SQL 오류를 내는 대신, 비교용 결과 표가 만들어지는 흐름이면 정상입니다.

## 6. 후속 분석이 실제로 실행되는지 확인하는 질문

아래 질문은 "답변만 설명하고 실제 분석은 안 하는 문제"를 점검할 때 좋습니다.

1. `오늘 XX공정에서 MODE별 생산량 알려줘`

### 기대 결과

- 단순 raw 데이터만 보여주는 것이 아니라
- MODE 기준 집계나 정리된 결과 표가 바로 나와야 합니다.

## 7. ENG'R 모드 확인

`ENG'R 모드`를 켠 뒤 같은 질문을 다시 해보면 아래를 더 볼 수 있습니다.

- pandas 처리 요약
- 생성된 코드
- 분석 과정 관련 추가 정보

반대로 끄면 실제 답변과 표 중심으로 더 간단하게 보여야 합니다.

## 8. 문제가 있으면 먼저 볼 파일

- UI가 안 뜨면: `app.py`
- 질문이 이상하게 해석되면: `core/parameter_resolver.py`
- 새 조회가 이상하면: `core/data_tools.py`
- LangGraph 흐름이 이상하면: `core/agent.py`
- 후속 pandas 분석이 이상하면: `core/data_analysis_engine.py`

## 9. 가장 짧은 스모크 테스트

시간이 없으면 아래 3개만 확인해도 됩니다.

1. `오늘 생산량 보여줘`
2. `상위 5개만 보여줘`
3. `어제 DA공정 생산량과 오늘 DA공정 생산량을 세부 공정별로 비교해줘`

이 세 개가 되면 보통 아래 흐름은 기본적으로 살아 있습니다.

- 새 조회
- 후속 분석
- LangGraph 다중 조회 비교
